# Production Test Dashboard v80 — Complete Dark-Control Visibility

## Purpose

v80 fixes the remaining dark-theme visibility problems reported in the Streamlit interface, especially:

- the help/question-mark icon appearing almost black on the dark sidebar;
- the **Choose wells** multiselect opening a white **No results** panel;
- weak contrast for multiselect tags, clear buttons, arrows, search cursors, and body-level dropdown menus.

The parser, WhatsApp ZIP handling, custom inactive-gap behavior, OCR workflow, continuous-test plotting, and themed exports remain unchanged from v79.

## Root causes

### 1. Help icon uses SVG strokes

Streamlit 1.58 renders the help icon as an SVG whose visible lines are controlled by `stroke`. Earlier CSS changed the button color but did not reliably override the SVG stroke generated by Streamlit, so the icon could remain dark on a dark background.

v80 directly styles the stable Streamlit selectors:

- `data-testid="stTooltipIcon"`
- `data-testid="stTooltipHoverTarget"`
- `svg.icon`

The icon now has a high-contrast stroke, visible circular control surface, hover state, and keyboard-focus ring.

### 2. BaseWeb no-results state is a separate list item

The multiselect dropdown is mounted in a body-level BaseWeb portal. Its empty state is rendered as:

- `li[aria-live="polite"][aria-atomic="true"]`

That node did not inherit all previous dropdown background rules, allowing Streamlit/BaseWeb's light background to remain visible in Dark mode.

v80 explicitly styles the popover surface, menu, listbox, options, selected/hovered states, and empty-state row using stable ARIA and BaseWeb attributes rather than generated class names.

## Dark-theme controls reviewed

The final high-contrast layer covers:

- help/question-mark icons and tooltip text;
- selectbox and multiselect surfaces;
- selected well/feature tags and remove icons;
- clear buttons and dropdown arrows;
- search text, placeholders, and caret;
- dropdown/popover surfaces;
- normal, hovered, and selected options;
- the **No results** empty state;
- date, time, number-input, popover, and menu icons;
- existing upload controls, expanders, checkboxes, toggles, scrollbars, alerts, dialogs, tables, and Plotly toolbar styling.
